# AI_AGENT_GUIDE.md

## Purpose
Explains how AI agents should work within this project.

AI agents must treat project documentation as the conceptual source of truth.

## Primary Project Sources (once created)

1. PROJECT_BRIEFING.md
2. DOMAIN_MODEL.md
3. CORE_PRINCIPLES.md
4. CONCEPT_MODEL.md
5. RELATIONSHIP_MODEL.md
6. SYSTEM_MODEL.md
7. ARCHITECTURE.md
8. IMPLEMENTATION_PLAN.md

If conflicts appear, conceptual documents override implementation.

## Working Rules

AI agents must:

1. Not invent undocumented domain concepts
2. Propose model updates before modifying architecture
3. Maintain consistent terminology
4. Update OPEN_QUESTIONS.md if uncertainty appears

## Model Change Proposals

If a conceptual change is required, the AI must create a proposal using PROPOSAL_TEMPLATE.md.

Concept Model changes require explicit confirmation.  
Architecture changes require explicit confirmation.  
Implementation changes do not.
