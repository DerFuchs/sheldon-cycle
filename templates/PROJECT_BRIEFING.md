# PROJECT_BRIEFING.md

## Purpose
Defines the high-level purpose, goals, and constraints of the project.

## Problem Statement
Describe the real-world problem the project addresses.

Focus on:
- existing friction
- who experiences the problem
- why current solutions are insufficient

Avoid describing implementation ideas.

## Goals
Describe the intended outcomes.

Examples:
- reduce complexity
- automate reasoning
- improve explainability
- reduce manual workflows

## Non-Goals
Explicitly list what the project does NOT attempt to solve.

## Success Criteria
How will we know the project succeeded?
