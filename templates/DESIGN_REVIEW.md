# DESIGN_REVIEW.md

## Purpose
Structured review checklist for the conceptual model.

## Review Checklist

### Domain Model
- Are entities clearly defined?
- Are real‑world constraints represented?

### Concept Model
- Are definitions precise?
- Are concept boundaries clear?

### System Model
- Does information flow logically?
- Are authority boundaries explicit?

### Architecture
- Does architecture reflect the conceptual model?
- Are responsibilities clearly separated?

### Common Failure Modes

- duplicated concepts
- ambiguous terminology
- hidden workflow logic
- accidental coupling
