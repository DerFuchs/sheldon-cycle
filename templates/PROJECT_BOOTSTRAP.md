# PROJECT_BOOTSTRAP.md
## Project Initialization Guide

This file initializes a new project using the Design‑OS workflow.

When starting a new project, follow these steps.

---

## Step 1 — Load Context

Read:

PROJECT_BRIEFING.md  
PROJECT_METHOD.md  
PROJECT_STRUCTURE.md  

These documents define the reasoning framework.

---

## Step 2 — Explore the Domain

Create or update:

DOMAIN_MODEL.md

Focus on:

- entities
- events
- constraints
- uncertainties

---

## Step 3 — Define Core Principles

Create:

CORE_PRINCIPLES.md

Principles act as design laws that guide architecture.

---

## Step 4 — Define Concepts

Create:

CONCEPT_MODEL.md

Each concept must include:

- definition
- purpose
- boundaries

---

## Step 5 — Model Relationships

Create:

RELATIONSHIP_MODEL.md

Describe how concepts interact.

---

## Step 6 — Simulate the System

Create:

SIMULATION.md

Test the model with:

- happy path
- ambiguous case
- failure case

---

## Step 7 — Derive Architecture

Create:

ARCHITECTURE.md

Architecture should emerge from the model.

---

## Step 8 — Plan Implementation

Create:

IMPLEMENTATION_PLAN.md

Define minimal viable system and phases.

---

## Sheldon Cycle (Design Iteration)

Observation
→ Model inconsistency
→ Proposal
→ Model refinement
→ Architecture adjustment
→ Implementation

Repeat until the system is conceptually coherent.
